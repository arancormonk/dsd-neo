# Issue Reporting

Use GitHub Issues for public defect reports and enhancement requests:

https://github.com/arancormonk/dsd-neo/issues

Use GitHub Discussions for broader usage questions when an issue is not yet
actionable:

https://github.com/arancormonk/dsd-neo/discussions

## Defect Reports

Include:

- DSD-neo version, tag, or commit
- operating system and compiler
- CMake preset or build command
- input source, output sink, protocol mode, and hardware backend when relevant
- minimal reproduction steps
- expected behavior
- actual behavior
- logs, test output, config snippets, or synthesized sample data when useful

Do not attach private captures, credentials, radio keys, API keys, or
machine-specific configuration. Redact or synthesize test data before sharing it
publicly.

## Security Reports

Do not report suspected vulnerabilities through public issues. Use the private
process in `SECURITY.md`.
